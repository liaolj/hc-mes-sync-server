# 项目总览 (Project Overview)

## 1. 文档目的
本文档旨在提供“合创 MES 数据同步服务器 (hc-mes-sync-server)”项目的全局视角，帮助新老团队成员快速了解项目的核心目标、技术栈与业务边界。

## 2. 项目描述
**hc-mes-sync-server** 是合创 MES 系统中的一个关键中间件/后端服务，用于**统一接收平坦度 (pt-sync) 与热阻 (rz-sync) 客户端上报的设备检测数据**，并提供数据存储、查询 API 以及轻量级的 Web 可视化管理界面。

该项目承载了生产车间设备边缘端数据向中心服务器汇聚的职责。

## 3. 核心功能域
* **设备数据上报接收**：提供基于 HTTP 的高并发数据接收接口（批量上传）。
* **终端设备(客户端)管理**：支持识别不同的数据采集工控机/设备（通过 `client_id`），并支持为设备设置可读备注名（Alias）。
* **业务数据持久化**：使用 SQLite (WAL模式) 提供无需额外部署的本地化、轻量级但是高性能的结构化数据存储引擎。
* **数据总览与查询**：提供内置 Web 数据看板与查询 API，支持按设备、条码筛选平坦度与热阻的测试数据。
* **数据清理能力**：支持按类型或者全量重置数据，用于测试及运维维护阶段。

## 4. 技术栈摘要
* **运行时**: Node.js 18+ (推荐 Node.js 20+)
* **Web 框架**: Express.js
* **数据库**: SQLite3 (`better-sqlite3` 驱动)
* **模板引擎**: EJS (用于服务端渲染数据看板)
* **进程守护**: PM2 (生产环境推荐)
* **CI/CD**: GitHub Actions (支持自动化部署至指定服务器)

## 5. 架构模式判断
属于**经典的轻量级单体架构 (Monolith) + 服务端渲染 (SSR) 混合模式**，同时也充当**数据接入网关 (Data Ingestion Gateway)** 的角色。整个项目没有复杂的分层，将路由、数据操作集中于少量文件中，强调“极简、快速部署、低运维成本”。

## 6. 部署方式判断
主要采纳**裸机部署 / 传统进程级部署**模式。
通过随源码提供的 `deploy.sh` 脚本和 PM2 在 Linux (Ubuntu/CentOS 等) 服务器上运行。环境依赖极少（仅需 Node.js 18+）。

## 7. 关联与对接系统
* **上游系统（数据生产者）**: 
  * `hc-mes-pt-sync-egui` (平坦度数据同步客户端桌面端应用)
  * `hc-mes-rz-sync-egui` (热阻数据同步客户端桌面端应用)
* **下游系统（数据消费者）**: 
  * MES 中心系统的其他模块，或工厂管理人员（通过内置的 Web 管理页面或者 API 访问）。
