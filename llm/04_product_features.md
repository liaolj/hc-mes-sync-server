# 产品功能介绍 (Product Features)

## 1. 文档目的
本文档旨在梳理 `hc-mes-sync-server` 所提供的主要业务功能与应用场景，适用于产品经理、测试人员及新加入的开发人员理解业务域。

## 2. 核心业务域与场景概述
本项目为“工业制造数据汇聚的中间层”。在合创 MES 一期生产线改造中，不同机台与工控机（分为平坦度测试机台与热阻测试机台）会持续产生离线的 CSV/Excel 检测数据。

本服务为这些工控机端提供一网统管的数据汇聚接口（利用 pt-sync 与 rz-sync 桌面客户端推送），并提供了轻量级的数据清洗后展示的功能。

## 3. 目标用户
* **系统服务 (M2M 通信)**: 分布在各大车间的上位机客户端，定时与本系统交互以上报检测数据。
* **现场工艺管理 / 运维人员**: 访问本系统提供的 Web 看板，抽查数据的上传及时率或在必要时修正机台代称。
* **MES 其他微服务**: 调取本系统的基础 API 拉取清洗整理好的结构化检测数据（或基于本系统直接将数据中转）。

## 4. 核心产品功能模块

### 4.1. 数据采集网关（API 核心能力）
系统拥有两套标准化的高并发 JSON 上报接口，用于接手并落盘数据：
- **【平坦度数据接收】**：接收带 `barcode`(条码)、`overall_area`(总面积)、`adhesive_area`(胶面积) 等核心字段的有效性分析平坦度记录。
- **【热阻数据接收】**：接收带 `barcode`(条码)、`channel`(通道)、`chip_temp`(芯片温度)、`thermal_resistance`(热阻值) 等数十个性能指标的热阻分析记录。

**功能特性**：
- 支持 **批量提交与合并写入**，使用事务包裹。
- 内置 **重复消解** 机制：强制根据 `[客户端身份, 源文件名称, 取样行号]` 组合约束做**幂等插入**（`INSERT OR IGNORE`），保证客户端在网络断开重试后的数据绝对幂等性。

### 4.2. 数据查询与分析引擎
系统允许对已存库的上百万数据通过丰富的筛选器查询。
- **支持维口**：支持基于“设备终端所属 Client ID” 与 “生产产品条码 (Barcode) 模糊匹配” 的查询方式。
- **内置分页**：在 API 及看板端原生支持基于偏移量的分页能力提取。

### 4.3. 客户端设备别名管理 (Client Management)
由于上报端设备的默认识别标识往往是 MAC 地址（例：`PC-001` 或网卡物理址），对人非常不友好。
- **设置与覆盖别名功能**：允许通过接口实时绑定 `client_id` 到业务意义的机器号（如“3号热阻线测试机台”）。系统内部会将其落盘到 `client_aliases` 独立映射表中。

### 4.4. 运营级 Web 数据报表看板
不需要打开复杂系统，服务直接开放出直观友好的轻量化 Web 管理面板（访问系统根 `/` 路由）。
- **实时统计看板**：呈现总数据汇聚量。
- **多 Tab 查询流**：平坦度和热阻两项独立记录分开管理查询，并允许即刻搜寻对应设备发出的记录追溯其是否含有问题。
- **别名回显映射**：利用内部联表关联逻辑展示机器原本可读业务名。

### 4.5. 风险运维管控 - 终态复位器
- **危险数据回滚重置 (Reset API)**: 供测试阶段或发生大规模数据污染时，使用专有受权重置整个业务库 `flatness` 和/或 `thermal` 的海量积累数据，直接将其还原至表结构空白状（常用于上线前全量脏压测数据的一次性清理）。
