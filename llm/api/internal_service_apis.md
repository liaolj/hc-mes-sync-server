# 内部服务通信接口 (Internal Service APIs)

## 结论说明
由于本项目架构十分简单（属于**单体应用**），**未引入**微服务架构，或者内部 gRPC 调用模式。

* 不存在如 GraphQL / RPC / Thrift 等额外的通信方式。
* 项目内部的所有路由调用均基于标准的 HTTP/Express。
* 无消息队列（如 RabbitMQ/Kafka）事件总线。所有的批处理“发布/消费”都在一个闭环生命周期（客户端发 HTTP 请求 => `server.js` 收到后同主线程 `sqlite` 入盘）中直接发生。
